## 候选确认 —— 改了什么
**保留你的真实模型**，只升级版式：

- **状态**：候选 / 已批准 / 已下发(sent) / 失败 / 已拒绝（沿用 `candidateStatusLabel`）
- **筛选链**：待确认 / 待补磁力 / 可批准 / 已下发 / 失败 / 已拒绝，下方独立一行 **来源**子筛选：全部来源 / 订阅 / 库存 / 补全 / 手动（带计数）
- **顶部汇总条**：大号「待确认数」+ 可批准 / 待补磁力 / 已下发 三个小指标（accent 辉光卡）
- **卡片**（取代扁平 task-card）：左侧大封面（带番号角标、可多选勾选）；中部 = 来源 pill + `已有 magnet`/`待补磁力` + 时间 + 标题(h3) + 原名 + 演员小头像·发行日 + （失败时）reason + （已下发时）`已关联任务 #id` + 详情/演员/补全 链接；右侧**决策区**：
  - 可批准：`N 种子·可批准` + **批准**(绿) + **策略处理**(ghost) + 填磁力/拒绝(quiet)
  - 待补磁力：红色左边框 + `待补磁力·暂不可批准` + **补磁力**(accent) + **填磁力** + 拒绝
  - 失败：**重试** + 拒绝；已下发：`已下发` badge；已拒绝：`已拒绝` + 恢复
- **批量条**：批准 / 按策略处理 / 批量拒绝（对应 bulk-restore/bulk-reject/process-visible）
- 保留你的：磁力编辑弹窗、候选详情弹窗(事件时间线)、CandidateRunPanel、分页 —— 原型未画，落地时沿用现有实现，套新卡片样式即可。

## 补全管理 —— 改了什么
结构与你代码一致：**演员选择 → 工作台（hero + 指标 + 四 tab）**

- **演员选择**：搜索「演员名/罗马音」+ 筛选；卡片 = 头像 + 编号 + 状态 pill（排队中/运行中/已完成/失败/待开始）+ 字段缺口数 + 选择。对齐 `ActorPickerView` / `ActorPortraitCard`。
- **工作台 hero**：大头像（封面色）+ 名字 + 编号 + 状态 pill + recent；右侧 **补全作品**（运行时转圈）/ **刷新版本条目**；topbar：更换演员 / 来源状态 / 全局队列。
- **指标**（stat-card）：补全来源 / 已匹配片库 / 补全新增 / 含版本条目（对齐 `workspaceMetrics`）。
- **四 tab**（对齐 `tabItems`：movies/jobs/diagnostics/sourceHealth）：
  - **作品字段**：工具栏 批量补详情 / 生成下载候选；筛选 匹配状态 + 质量 + 番号搜索；**字段缺口 ledger**（总数 + 每字段「N 个缺口/已齐」）；影片行 = 封面 + 番号/标题/发行日·时长·片商·分类 + **field-chip 网格**（字段名 + 值 / 缺失）+ 匹配状态 pill + 补详情 / 诊断。对齐 `movieFieldChips` / `movieMatchLabel`。
  - **任务队列**：任务汇总 ledger（运行中/排队中/已完成/失败）+ job 卡（头像 + 标签 + #id·时间 + attempt 摘要 + 状态 pill）。对齐 `SupplementJobList`。
  - **来源诊断**：launchpad 三卡（待确认候选 / 字段缺口 / 可补详情）+ 诊断入口焦点影片（badges：候选待确认 / N 字段缺口 / 可追溯来源 / 字段已齐）+ 待诊断作品轨 + 逐字段来源候选（采用/改用）。对齐 `RepairLaneTab` 的 `diagnosticsLaunchpadRows` / `diagnosticsFocusMovieBadges` + SourceDiagnosticsDialog 的 匹配/取消匹配/忽略。
  - **来源状态**：汇总（接入/可参与补全/降级/隔离）+ 来源行（连续失败 / 最近错误 / 下一步）+ 隔离 runbook。对齐 `SourceHealthPanel`（可参与字段补全 / 隔离模式 暂停 24h）。
