## P1-2 · 详情弹窗演员区就地订阅（招牌闭环）

### 现状
`components/VideoModal.vue` 的"演员"区（约 149–172 行）每位演员只有头像 + 名字 + 点击跳转，**没有订阅按钮**。
要订阅得跳去 `/subscription` 再搜一次——信息与动作割裂。

### 现成地基（不要重写，直接用）
- **`utils/subscriptionState.js`** —— 全局 reactive 真相源，已具备：
  - `subscriptionState.init()` / `isSubscribed(actressId)` / `toggle(actressId, actressName)`（内部已调
    `api.toggleSubscription` / `deleteSubscription`，已 `refresh()`，已向 `listeners` 广播变更）；
  - `subscriptionState.subscribe(cb)` 订阅变更、`count`（reactive 计数）。
  → **这意味着 Subscription 页 / Today / 弹窗任意一处订阅，别处自动同步**（P3-3 已被它天然满足）。
- **`utils/displayLang.js` `displayName()`** —— 翻译名优先显示规则（弹窗已在用 `actressNameDisplay`）。

### 改动（仅 `VideoModal.vue` 一个文件）
1. **演员区每项加订阅按钮**。把跳转点击从整块挪到头像/名字上（避免点订阅误触跳转），按钮独立：
   ```html
   <div class="actress-avatar-item" v-for="actress in video.actresses" :key="actress.id">
     <div class="actress-avatar clickable" @click="$emit('navigate', { type: 'actress', item: actress })"> … </div>
     <div class="actress-name clickable" @click="$emit('navigate', { type: 'actress', item: actress })" v-html="actressNameDisplay(actress)"></div>
     <button
       class="actress-sub-btn"
       :class="{ 'is-subscribed': isSubscribed(actress.id), 'is-busy': subBusy.has(String(actress.id)) }"
       :disabled="!actress.id || subBusy.has(String(actress.id))"
       :aria-pressed="isSubscribed(actress.id)"
       @click.stop="toggleActressSub(actress)"
     >{{ isSubscribed(actress.id) ? '已订阅' : '订阅' }}</button>
   </div>
   ```
   > 样式：**复用现有订阅按钮的视觉语言**——参照 `ActorPortraitCard.vue` 的 `.actor-portrait-card__subscribe` /
   > `.is-subscribed` 规则（绿=已订阅、灰=可订阅、disabled 态），用同一批 token，别新造配色。
2. **脚本接线**（`VideoModal.vue` `<script>`）：
   ```js
   import { subscriptionState } from '../utils/subscriptionState.js'
   // data(): subBusy: new Set()
   // created()/mounted(): subscriptionState.init().catch(() => {})  // 幂等，失败不致命
   methods: {
     isSubscribed(id) { return id ? subscriptionState.isSubscribed(id) : false },
     async toggleActressSub(actress) {
       const id = actress?.id; if (!id) return
       const key = String(id)
       if (this.subBusy.has(key)) return
       this.subBusy.add(key); this.subBusy = new Set(this.subBusy)   // 触发响应式
       const wasSubscribed = this.isSubscribed(id)
       try {
         await subscriptionState.toggle(id, this.displayName(actress, 'name_kanji', 'name_romaji'))
         this.$message?.success?.(wasSubscribed ? '已取消订阅' : '已订阅')
       } catch (e) {
         this.$message?.error?.(apiErrorMessage(e, '订阅操作失败'))
       } finally {
         this.subBusy.delete(key); this.subBusy = new Set(this.subBusy)
       }
     },
   }
   ```
   > `subscriptionState.toggle` 内部已 `refresh()` 并广播，registry 是 reactive，按钮文案/态会自动更新——
   > **不需要手写乐观更新回滚**，toggle 失败时 registry 未变、按钮自然保持原态。`subBusy` 仅防连点。
3. **男优区不加订阅**（无对应订阅模型），保持现状。

## P3 · 健壮性打磨（有余力做，按价值排序）

### P3-1 Today 各区块错误态可重试（替代静默空）
- **现状**：`views/Today.vue` 多个 loader `.catch` 后置空数组，整页失败只看到空，分不清"真没有"还是"加载失败"。
- **改**：每个 loader 记录自己的 `xxxError` 标志；模板里该区块 `v-if="xxxError"` 显示一行
  `<AppleErrorState>`（项目已有组件）带"重试"，点击只重拉**该区块**的 `loadXxx()`，而非整页 `loadAll`。
- **落点**：`views/Today.vue` + 复用 `components/AppleErrorState.vue` / `AppleEmptyState.vue`。
- **验收**：断网时各区块显示可点重试的错误条；恢复后单块重拉成功；空数据仍显示空态（与错误态区分）。

### P3-2 hero "最近查看" 失效回退
- **现状**：hero 从 localStorage 读"上次点开的一部"；若该番号已被删/不存在，hero 可能卡在死数据上。
- **改**：`loadHero` 读 localStorage 后做一次有效性校验（带 `savedAt` 时间戳，过期或 `content_id` 取详情 404
  时丢弃），回退到"最新入库第一部"（现有回退路径）。
- **落点**：`views/Today.vue`、`HERO_STORAGE_KEY` 相关。

### P3-3 订阅/收藏全局一致 —— ✅ 已天然满足
- `subscriptionState` / `favoriteState` 都是 reactive 单例 + listener 广播。P1-2 直接消费 `subscriptionState`
  即自动满足"一处订阅、别处同步"。**无需额外工作**，仅在验收 P1-2 时确认 Subscription 页用的是同一个 state（它本就是）。
