# 下载源管理与单片来源选择设计

**日期：** 2026-07-14

## 背景

下载中心的“下载源”页面目前把在线 M3U8、单个 Torznab 配置和 AVDB 状态全部平铺展示。Torznab 表单即使尚未保存也占据页面，AVDB 也在未添加时直接出现。这与相邻的“下载器”管理体验不一致，也没有把“内置能力”和“用户已添加配置”区分开。

后端实际上已经能从 `sources.torznab` 和 `sources.torznab_instances` 注册多个 Torznab 来源，但前端只读取和保存 `sources.torznab`。影片弹窗则只展示 JavInfo 已返回的磁力，点击后直接下发到默认下载器；用户配置的 Torznab 和 AVDB 只参与候选补磁力的自动检索，尚不能用于单片影片的显式来源选择。

## 目标

- 将下载源页面改造成与下载器一致的工具栏、已保存列表和新增/编辑弹窗模式。
- 始终保留内置 HLS，且不要求用户添加或配置。
- 支持添加多个 Prowlarr、Jackett 或通用 Torznab 实例。
- 仅在用户确认并且后端保存成功后，把新来源加入列表。
- 将 AVDB 设计为需要用户显式添加的单例，并保留首次同步完成后才能启用的门槛。
- 在单片影片磁力区提供“自动”默认项和指定来源选择。
- 自动查询沿用批量补全已有的并行搜索、评分和优选语义。
- 保持批量补全现有“审核 / 自动”策略不变。

## 非目标

- 不把下载源扩展成第三方插件市场或动态安装框架。
- 不允许用户修改 AVDB 上游仓库地址或其他受安全约束的内部参数。
- 不把 HLS 和磁力索引混成同一种下载协议；M3U8 下载继续使用独立入口。
- 不重做下载器选择或下载任务创建流程；选择磁力后仍沿用当前默认下载器行为。
- 不改变批量候选的策略、限额、审核状态或自动调度规则。
- 不要求保存 Torznab 前执行网络探活；连接测试可作为后续独立功能。

## 领域模型

### 内置 HLS

HLS 是内置在线播放来源，始终作为下载源列表第一行出现。它标记为“内置 / 可用”，不提供新增、编辑、停用或删除操作。现有在线播放和 M3U8 下载入口保持原样。

### Torznab 来源

每个 Prowlarr、Jackett 或通用 Torznab 配置都是独立来源，包含稳定 ID、唯一显示名称、提供方类型、启用状态、服务地址、API Key、Indexer、Categories、返回上限和超时。允许保存多个实例，并按用户列表顺序注册和自动查询。

API Key 永远不返回明文。读取时只返回 `api_key_configured`；编辑时空白 API Key 表示保留现值，新建时必须提供 API Key，除非后端已配置 `JAVHUB_TORZNAB_API_KEY` 环境变量。

### AVDB 来源

AVDB 是 ID 固定为 `avdb` 的单例。是否出现在管理列表由 `sync_enabled` 表示：未添加时 `sync_enabled=false`，添加成功后为 `true`。`enabled` 仅表示它是否参与磁力检索。

首次添加会保存 `sync_enabled=true`、`enabled=false`，然后跳转到 `avdb_sync` 系统作业。只有后端状态确认存在可用索引后才允许启用。移除 AVDB 会设置 `enabled=false`、`sync_enabled=false` 并停止定时检查，但保留本地索引数据；以后重新添加时若已有有效索引，可以直接启用而不强制重复全量同步。

## 后端接口

新增一组 `/api/v1/sources` 管理接口，与现有 `/api/v1/sources/health` 共用来源领域但职责分离：

- `GET /api/v1/sources/config`：返回内置 HLS、已保存 Torznab 列表、AVDB 配置/状态摘要和可新增类型。
- `POST /api/v1/sources`：新增 Torznab 或 AVDB，成功后返回最新来源快照。
- `PUT /api/v1/sources/{source_id}`：编辑名称、配置或启用状态，成功后返回最新来源快照。
- `DELETE /api/v1/sources/{source_id}`：移除 Torznab，或停用并移除 AVDB 配置，成功后返回最新来源快照。
- `POST /api/v1/sources/search`：按番号和 `source_id` 搜索磁力；`source_id=auto` 表示所有已启用磁力来源。

请求使用 Pydantic 模型约束来源类型、URL、整数范围和必填字段。Torznab 名称必须去除首尾空白后非空且唯一；稳定 ID 由后端创建，编辑名称不会改变 ID。AVDB 重复添加返回明确的冲突错误。

配置写入由专用来源配置服务负责，不能直接依赖通用 `/config` 的列表替换语义。该服务按稳定 ID 保留未提交的新 API Key，原子写回 YAML，再调用 `register_all_sources()`；AVDB 变化还会刷新调度作业。

## 配置兼容

读取时继续兼容现有单例 `sources.torznab` 和列表 `sources.torznab_instances`：

- 旧 `sources.torznab` 在管理 API 中映射为确定性的兼容 ID。
- 已有 `torznab_instances` 如果没有稳定 ID，读取时生成确定性的兼容 ID，首次成功写入后持久化 ID。
- 新增来源写入 `torznab_instances`，不破坏旧单例配置。
- 删除旧单例时将其重置为禁用的空默认值；不会误删其他实例。
- 运行时归一化只向 `TorznabSource` 传递搜索所需字段，管理元数据不会泄漏到构造函数。

配置 GET 和导出继续递归移除所有 API Key。来源管理 GET 额外返回布尔型的密钥已配置状态，不暴露密钥内容。

## 下载源管理界面

### 工具栏与列表

`IndexerSourcePanel` 改为下载器面板同级的管理表面：顶部显示“下载源”、默认自动选择说明和启用数量，右侧提供刷新与新增图标按钮。列表依次显示：

1. 固定的在线 M3U8 内置行；
2. 用户已保存的 Torznab 来源行；
3. 用户已添加的 AVDB 行。

未保存的 Torznab 和未添加的 AVDB 不显示为空壳行。每个 Torznab 行显示提供方、服务主机、启用状态以及编辑、启停、删除操作。AVDB 行显示未同步、同步中、可用或失败状态，并提供启停、查看系统作业和移除操作。底部显示来源总数与启用数，不设置另一个全局保存按钮。

### 新增/编辑弹窗

新增按钮打开与下载器编辑器一致的 sheet。第一步选择“磁力索引”或“AVDB 公开库”；AVDB 已添加时该选项禁用并说明原因。

磁力索引表单包含名称、提供方类型、服务地址、API Key、Indexer、Categories、返回上限、超时和启用开关。新建主按钮为“保存并添加”，编辑时为“保存更改”。前端做必填和范围校验，后端保存失败时保持弹窗和输入不变；只有收到成功快照后才关闭并更新列表。

AVDB 弹窗说明固定公开库、首次同步流程和磁盘/时间成本，不提供任意仓库地址。主按钮为“添加并前往首次同步”；保存成功后显示 AVDB 行，并跳转 `/system-jobs?job=avdb_sync&return_to=/downloads?tab=indexer&source=avdb`。如果本地已有可用索引，按钮改为“添加来源”，保存后停留在当前页面。

### 启停与移除

启停和移除均为即时持久化操作。请求进行中禁用该行重复操作；失败时恢复原状态并显示错误。删除 Torznab 和移除 AVDB 使用确认对话框。HLS 不显示这些操作。

## 单片影片来源选择

`VideoMagnetSection` 在磁力列表标题旁增加来源菜单和“查找磁力”命令。菜单默认选择“自动”，其余选项来自已启用 Torznab 实例和已启用、可用的 AVDB。HLS 不出现在磁力菜单中，因为它不返回磁力；原有 M3U8 下载按钮继续独立存在。

打开影片弹窗不会自动触发外部查询，避免每次浏览详情都消耗索引器请求。用户点击“查找磁力”后：

1. 使用影片 `dvd_id`，缺失时回退 `content_id`，调用来源搜索接口。
2. “自动”模式并行查询所有已启用来源；指定模式只查询所选来源。
3. 后端复用候选补磁力的代码匹配、评分和排序逻辑。
4. 前端把返回结果与影片已有磁力合并，优先按 info hash 去重，没有 hash 时按规范化磁力 URI 去重。
5. 每一行显示来源名称、标题、大小、清晰度、字幕和做种信息；用户选择具体磁力后再点击现有下载按钮。

来源搜索错误在磁力区内显示，不关闭影片弹窗。自动模式允许部分成功：单个来源失败不丢弃其他来源结果，并在结果摘要中说明失败来源。没有启用的磁力来源时，提供前往下载源管理的明确入口。

## 批量补全行为

候选和补全的现有审核/自动分工保持不变：

- 自动模式继续调用所有已启用来源并通过现有评分选择最佳磁力。
- 审核模式继续逐条展示候选、磁力来源和备选磁力，由用户确认后下发。
- 单片来源菜单只是一次请求的临时覆盖，不修改全局启用状态，也不影响定时自动处理。

为了避免单片搜索与批量搜索产生不同结果，候选处理器中的搜索、代码校验、评分和去重逻辑会提取到共享服务；候选处理器和新来源搜索接口共同调用该服务。

## 状态与错误处理

- 来源列表加载失败时保留上次成功数据，并提供重试。
- 新增/编辑校验失败时在字段附近显示具体原因，不提交请求。
- 保存或删除失败时不改变可见列表；启停失败时回滚开关。
- API Key 留空编辑不会清除已有密钥；明确移除密钥不在本次范围。
- AVDB 状态未知时禁止启用、移除和首次同步跳转之外的破坏性状态修改。
- 搜索请求防止同一影片重复并发；切换影片或关闭弹窗后忽略过期响应。
- 自动搜索保留成功来源的结果，并返回结构化的来源错误摘要。

## 组件边界

- `IndexerSourcePanel.vue`：只负责来源列表、弹窗和用户事件，不直接调用 HTTP。
- `Downloads.vue`：加载来源快照，执行新增、编辑、启停、删除和 AVDB 导航。
- 新的来源展示辅助模块：默认草稿、类型标签、摘要、去重和状态映射等纯函数。
- `VideoMagnetSection.vue`：展示来源菜单、搜索状态和合并后的磁力结果。
- `VideoModal.vue`：提供影片标识、持有一次弹窗会话的搜索状态并处理过期响应。
- 后端来源配置服务：持久化、多实例兼容、密钥保留和注册刷新。
- 后端磁力搜索服务：指定/自动来源查询、共享评分、排序和错误汇总。

## 测试策略

实施遵循测试优先：

1. 后端配置测试覆盖旧单例、多实例稳定 ID、唯一名称、列表删除和 API Key 保留。
2. 路由测试覆盖来源快照、新增/编辑/启停/删除、AVDB 单例与首次同步门槛。
3. 搜索服务测试覆盖自动并发、指定来源、部分失败、代码匹配、排序和去重。
4. 回归测试证明候选自动补磁力仍使用相同共享搜索逻辑和评分结果。
5. 前端组件契约测试覆盖 HLS 固定行、仅显示已保存来源、弹窗保存门槛、AVDB 添加流程和操作失败回滚。
6. 影片磁力测试覆盖“自动”默认、动态来源菜单、单源请求、合并去重、来源标签、空状态和过期响应隔离。
7. 运行完整后端测试、前端 `npm run check`、生产构建和 `git diff --check`。
8. 使用托管服务在桌面和移动视口验证下载源新增/编辑、AVDB 首次同步返回以及单片来源选择流程。

## 验收标准

- 下载源页面首屏不再平铺未保存的 Torznab/AVDB 配置表单。
- HLS 始终存在且不可编辑；未添加的 AVDB 不显示在列表中。
- 可以保存多个唯一命名的 Torznab 来源，刷新页面后配置仍在且密钥未丢失。
- 新来源只在后端保存成功后出现，编辑、启停和删除失败不会留下错误的本地状态。
- 首次添加 AVDB 后进入系统作业，首次同步未完成前不能启用。
- 单片影片默认选择“自动”，也能指定任一已启用磁力来源查询。
- 自动和指定来源结果正确标注来源、排序并去重，现有影片磁力仍可直接使用。
- 批量审核/自动策略、HLS 下载、下载器配置和下载任务 API 没有行为回归。
- 前后端完整测试、构建及桌面/移动浏览器验收通过。
