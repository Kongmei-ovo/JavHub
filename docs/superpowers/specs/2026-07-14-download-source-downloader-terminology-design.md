# 下载中心“下载器 / 下载源”术语校正设计

**日期：** 2026-07-14

## 背景

下载中心当前把第二个工作区标签及其客户端管理面板称为“下载源”，同时把第三个影片来源工作区称为“来源管理”。这使两个不同概念发生错位：115 Open、qBittorrent、Transmission 和 Aria2 是负责接收任务的下载器；在线 M3U8、Torznab 和 AVDB 才是提供播放地址或磁力链接的下载源。

## 目标

- 将下载中心第二项及所有客户端管理相关文案统一为“下载器”。
- 将下载中心第三项统一命名为“下载源”。
- 修正任务下发、任务列表和返回导航中沿用错误语义的文案。
- 保留资料补全、磁力检索等真正表示内容来源的“下载源”用词。

## 术语映射

| 使用场景 | 当前文案 | 调整后文案 |
| --- | --- | --- |
| 下载中心第二个标签 | 下载源 | 下载器 |
| 下载器管理标题、操作、空态、计数、编辑器和保存提示 | 下载源 | 下载器 |
| 任务列表的默认目标 | 默认下载源 | 默认下载器 |
| 候选任务的下发目标 | 默认下载源 / 下载源 | 默认下载器 / 下载器 |
| 下载中心第三个标签 | 来源管理 | 下载源 |
| 系统作业返回第三项 | 返回来源管理 | 返回下载源 |

## 实现边界

本次只修改用户可见文案、按钮的 `title` / `aria-label` 以及对应的前端契约测试。`downloaders`、`indexer` 等路由查询值、组件名、变量名和 API 数据结构保持不变，现有深链和后端接口不受影响。

`IndexerSourcePanel` 中“影片来源”“磁力索引”“启用来源”等文案语义正确，保持不变。资料补全阶段、后端来源插件和磁力查找流程中的“下载源”同样保持不变。

## 测试与验证

1. 先添加文案契约测试，要求三个工作区标签依次表达“下载任务 / 下载器 / 下载源”，并覆盖下载器面板的标题、操作、空态、计数和编辑器文案。
2. 运行目标测试并确认它因旧文案而失败。
3. 最小化修改生产文案后，重新运行目标测试。
4. 运行前端完整测试与生产构建，并执行 `git diff --check`。
5. 使用已托管的前端服务检查 `/downloads?tab=downloaders` 和 `/downloads?tab=indexer`，确认标签、面板及辅助文案显示正确。

## 验收标准

- 下载中心第二项显示“下载器”，其中不再出现把客户端称为“下载源”的文案。
- 下载中心第三项显示“下载源”，系统作业返回按钮与该名称一致。
- 任务目标和候选下发目标均称为“下载器”。
- 真正表示影片、在线播放或磁力来源的文案没有被误改。
- 路由、接口和配置行为没有变化，相关前端测试及构建通过。
