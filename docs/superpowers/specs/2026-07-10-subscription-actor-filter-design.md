# 演员订阅列表检索设计

## 目标

在“演员订阅”页面为当前已订阅演员提供即时检索，使演员数量增长后仍能快速定位目标，并与“资料补全”页面的“筛选订阅演员”交互和视觉语言保持一致。

## 范围

- 只筛选当前已经加载到“演员订阅”页面的订阅记录。
- 不搜索或展示尚未订阅的演员；添加新演员仍使用现有“添加演员”入口。
- 不新增后端接口，不引入拼音转换、编辑距离或服务端分页。

## 交互设计

- 在订阅卡片内容区顶部放置一个全宽检索框，提示文案和无障碍标签均为“筛选订阅演员”。
- 用户输入时立即在前端过滤，无需按回车或等待网络请求。
- 匹配不区分拉丁字母大小写，并采用部分字符串匹配；输入姓名的一部分即可命中。
- 检索框有内容时显示清除按钮；清除后恢复完整订阅列表。
- 有订阅但没有匹配项时，显示“没有匹配演员”空状态，说明当前关键词没有匹配到订阅演员，并提供“清除筛选”操作。
- 页面本身没有任何订阅时，继续显示现有“还没有订阅任何演员”空状态和添加入口。
- 顶部“已订阅 / 候选 / 待补磁力”统计始终反映全部订阅，不随筛选结果变化。

## 匹配字段

每条订阅由订阅记录和已加载的演员元数据共同构造检索文本，覆盖：

- 中文译名：`name_kanji_translated`、`name_romaji_translated`
- 日文姓名：`name_kanji`
- 假名：`name_kana`
- 罗马音：`name_romaji`
- 订阅记录中的演员名称：`actress_name`
- 演员 ID：`actress_id`

缺少元数据时仍使用订阅记录中的名称和演员 ID 匹配；元数据异步补齐后，筛选结果随响应式状态自动更新。

## 编辑模式

- 卡片选择、批量暂停、恢复、自动策略和取消订阅等现有能力保持不变。
- 编辑模式下只展示符合当前关键词的演员。
- “全选”和“是否已全选”的判断只针对当前可见的筛选结果，避免批量选中被检索隐藏的演员。
- “清空”仍清除全部已选项；切换或清除关键词不会自动取消此前选择。
- 批量操作摘要继续显示实际已选择数量，并额外以当前页面已有文案表达全部订阅数量，不把筛选结果误称为订阅总数。

## 组件与数据流

- `Subscription.vue` 新增检索关键词状态和 `filteredSubscriptions` 计算值。
- 卡片循环改为使用 `filteredSubscriptions`；总数统计和数据加载仍使用原始 `subs`。
- 检索文本由一个小型纯函数生成，便于独立测试多语言字段、ID 和大小写匹配。
- 检索框沿用订阅功能样式表中已有玻璃控件令牌，并在订阅内容区补充与“资料补全”一致的尺寸、焦点态和清除按钮布局。

## 错误与边界状态

- 元数据加载失败不阻断检索，订阅名称和 ID 仍可用。
- 仅输入空白等同于没有筛选。
- 没有匹配项不是加载错误，不显示重试或网络错误文案。
- 加载期间继续显示现有骨架屏；加载完成后再展示检索结果或对应空状态。
